2.1 File xOR


We will again be using the xOR encryption/decryption, only with files instead of plaintext. What
this encryption function does is it takes a txt file as its input, an empty output txt file, and the key
generates a key of bits the same length, and creates a cipher by combining the two pieces of bit
that will be used to encrypt the input file.

Then, again using exclusive or plus the key, the input file is turned into a cipher text file. This is
done by populating the empty output file with the cipher.

This function can also be modified to work for any file type.
